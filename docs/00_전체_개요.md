# 00. 전체 개요

이 문서는 F1TENTH 레포의 전체 구조와 실험 흐름을 빠르게 이해하기 위한 문서다.

---

## 1. 이 레포의 목적

이 레포는 F1TENTH 차량을 ROS2 기반으로 구동하기 위한 race stack이다.

주요 목적은 다음과 같다.

- 차량 하드웨어 bringup
- LiDAR 데이터 수신
- VESC 기반 모터/서보 제어
- 수동 조작
- 웹 대시보드 조작
- SLAM 맵 생성
- 저장된 맵 기반 위치추정
- Pure Pursuit 경로추종
- Stanley Controller 기반 장애물 회피
- 실험자가 이해하기 쉬운 한국어 문서화

---

## 2. 전체 실험 흐름

```text
빌드
  ↓
환경 적용
  ↓
차량 bringup
  ↓
수동 조작 또는 웹 조작 확인
  ↓
RViz2로 센서와 TF 확인
  ↓
SLAM으로 맵 생성
  ↓
맵 저장
  ↓
저장된 맵으로 위치추정
  ↓
Pure Pursuit 또는 Stanley 기반 주행
```

---

## 3. scripts 기준 실행 순서

```bash
bash scripts/01_build.sh
source install/setup.bash
bash scripts/02_bringup.sh
bash scripts/03_web_dashboard.sh
bash scripts/04_rviz.sh
bash scripts/05_slam_start.sh
bash scripts/06_save_map.sh maps/saved/my_map
bash scripts/07_localization.sh
bash scripts/08_pure_pursuit.sh
bash scripts/09_stanley_avoidance.sh
```

| 스크립트 | 역할 |
|---|---|
| `01_build.sh` | colcon build 실행 |
| `02_bringup.sh` | 차량 하드웨어 bringup |
| `03_web_dashboard.sh` | 웹 대시보드 실행 |
| `04_rviz.sh` | RViz2 실행 |
| `05_slam_start.sh` | SLAM 실행 |
| `06_save_map.sh` | 생성한 맵 저장 |
| `07_localization.sh` | 저장된 맵 기반 위치추정 실행 |
| `08_pure_pursuit.sh` | Pure Pursuit 주행 실행 |
| `09_stanley_avoidance.sh` | Stanley 장애물 회피 주행 실행 |

## 4. 주요 ROS2 토픽

| 이름 | 의미 |
|---|---|
| `/drive` | 차량 구동 명령 |
| `/webop` | 웹 대시보드 조작 명령 |
| `/teleop` | 수동 조작 명령 |
| `/scan` | LiDAR 스캔 |
| `/odom` | 차량 odometry |
| `/map` | 맵 |
| `/tf` | 좌표계 변환 |
| `/tf_static` | 고정 좌표계 변환 |

---

## 5. 주요 frame 이름

| frame | 의미 |
|---|---|
| `map` | 전역 맵 좌표계 |
| `odom` | odometry 좌표계 |
| `base_link` | 차량 기준 좌표계 |
| `laser` | LiDAR 센서 좌표계 |

## 6. 주요 패키지 역할

| 패키지/폴더 | 역할 |
|---|---|
| `src/f1tenth_system/` | 차량 bringup, VESC, teleop, mux 등 |
| `src/particle_filter/` | 맵 기반 위치추정 |
| `src/pure_pursuit/` | raceline 기반 경로추종 |
| `src/safety_node/` | 충돌 위험 시 정지 |
| `src/stanley_avoidance/` | Stanley 제어와 장애물 회피 |
| `src/state_estimation/` | 상태추정, odom/IMU 융합 관련 기능 |
| `mission_control/` | 웹 대시보드와 보조 조작 도구 |
| `scripts/` | 실험 명령 단순화 |
| `docs/` | 한국어 문서 |

## 8. 이 단계에서 하지 않는 것

현재 문서화 단계에서는 아래 작업을 하지 않는다.

- ROS2 패키지 이름 변경
- launch 파일명 변경
- config YAML 파일명 변경
- topic 이름 변경
- frame 이름 변경

이 단계의 목적은 구조를 바꾸는 것이 아니라, 기존 구조를 한국어로 이해하기 쉽게 정리하는 것이다.
