# 09. 실행 묶음과 주행 모드

이 문서는 F1TENTH 레포에서 여러 기능을 어떤 조합으로 실행해야 하는지 정리한다.

단순히 `pure_pursuit` 코드 하나, `stanley_avoidance` 코드 하나를 실행하는 것이 아니라, 실제 주행에서는 여러 노드가 한 묶음으로 함께 실행되어야 한다.

---

## 1. 핵심 요약

F1TENTH 실험은 크게 아래 실행 묶음으로 나눌 수 있다.

| 목적 | 실행 묶음 |
|---|---|
| 차량 센서와 하드웨어만 켜기 | Bringup 묶음 |
| 웹에서 직접 조작하기 | Bringup + Web Dashboard |
| 맵 만들기 | Bringup + RViz2 + SLAM + 수동조작 |
| 저장된 맵에서 위치 찾기 | Bringup + RViz2 + Localization |
| Pure Pursuit로 경로 따라가기 | Bringup + RViz2 + Localization + Pure Pursuit |
| Stanley로 장애물 회피 주행하기 | Bringup + RViz2 + Localization + Stanley Avoidance |

가장 중요한 점은 다음과 같다.

```text
Pure Pursuit와 Stanley Avoidance는 단독으로 실행하는 코드가 아니다.
둘 다 차량 bringup, 센서, TF, 맵, localization이 먼저 준비되어야 한다.
```

---

## 2. 전체 구조

전체 실행 구조를 크게 보면 아래와 같다.

```text
공통 준비 묶음
  ├─ 차량 bringup
  ├─ LiDAR /scan
  ├─ odom /odom
  ├─ TF /tf
  ├─ RViz2
  └─ localization

그 다음 선택
  ├─ Pure Pursuit 경로추종
  └─ Stanley Avoidance 장애물 회피
```

Pure Pursuit와 Stanley의 차이는 마지막 제어기 부분이다.

```text
Pure Pursuit:
  현재 위치 + waypoint/raceline
  → 목표점 추종
  → /drive 발행

Stanley Avoidance:
  현재 위치 + waypoint/raceline + /scan
  → Stanley 경로추종
  → 장애물 회피 보정
  → /drive 발행
```

## 3. Bringup 묶음

Bringup은 차량 하드웨어와 센서를 살리는 가장 기본 실행 묶음이다.

### 실행 목적

Bringup은 다음 기능을 실행하기 위한 단계다.

- VESC 연결
- LiDAR 연결
- `/scan` 발행
- `/odom` 발행
- `/tf` 발행
- mux 실행
- 수동조작 입력 준비
- 최종 `/drive` 명령 전달 준비

### 실행 명령

터미널 1:

```bash
cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/02_bringup.sh
```

### 확인 명령

다른 터미널에서 확인한다.

```bash
cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

ros2 topic list
```

확인할 주요 토픽:

```text
/scan
/odom
/tf
/tf_static
/drive
/webop
/teleop
```

LiDAR 확인:

```bash
ros2 topic echo /scan
```

odom 확인:

```bash
ros2 topic echo /odom
```

TF 확인:

```bash
ros2 run tf2_tools view_frames
```

## 4. 웹 수동조작 묶음

웹 수동조작 묶음은 차량을 직접 조작할 때 사용한다.

### 실행 묶음

```text
Bringup
  ↓
Web Dashboard
  ↓
브라우저 조작
  ↓
/ webop
  ↓
mux
  ↓
/drive
```

정확한 토픽 흐름은 레포의 mux 설정에 따라 다를 수 있지만, 기본 개념은 웹 대시보드가 `/webop` 명령을 만들고 최종적으로 차량 주행 명령으로 이어지는 것이다.

### 터미널 1: bringup

```bash
cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/02_bringup.sh
```

### 터미널 2: 웹 대시보드

```bash
cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/03_web_dashboard.sh
```

브라우저 접속:

```text
http://localhost:5000
```

다른 PC나 스마트폰에서 접속할 경우:

```text
http://노트북_IP:5000
```

### 확인 명령

웹 대시보드에서 `전진`, `정지`, `좌회전`, `우회전` 버튼을 눌렀을 때 `/webop`가 바뀌는지 확인한다.

```bash
ros2 topic echo /webop
```

최종 drive 명령 확인:

```bash
ros2 topic echo /drive
```

### 주의

처음 테스트할 때는 다음 값을 낮게 설정한다.

```text
최대 속도: 0.1 ~ 0.2
최대 조향: 0.10 ~ 0.20
```

## 6. Localization 묶음

Localization은 저장된 맵 위에서 차량의 현재 위치를 추정하는 단계다.

Pure Pursuit와 Stanley Avoidance를 실행하기 전에 반드시 필요하다.

### 실행 묶음

```text
Bringup
  ↓
RViz2
  ↓
Localization
  ↓
RViz2에서 현재 위치 확인
```

### 터미널 1: bringup

```bash
cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/02_bringup.sh
```

### 터미널 2: RViz2

```bash
cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/04_rviz.sh
```

### 터미널 3: localization

```bash
cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/07_localization.sh
```

### RViz2에서 확인할 것

- Fixed Frame이 `map`인지 확인
- 저장된 맵이 보이는지 확인
- `/scan`이 맵의 벽과 대체로 겹치는지 확인
- 차량 위치가 실제 차량 위치와 비슷한지 확인
- particle 또는 inferred pose가 정상적으로 보이는지 확인

초기 위치가 틀리면 RViz2의 `2D Pose Estimate`를 사용해 차량 위치를 다시 찍는다.

## 7. Pure Pursuit 주행 묶음

Pure Pursuit는 저장된 맵에서 위치추정이 된 상태에서 waypoint 또는 raceline을 따라가는 주행 방식이다.

### 실행 묶음

```text
Bringup
  ↓
RViz2
  ↓
Localization
  ↓
Pure Pursuit
  ↓
/drive
  ↓
VESC
  ↓
차량 주행
```

### 필요한 준비물

```text
[ ] 차량 bringup이 정상 동작한다.
[ ] /scan이 나온다.
[ ] /odom이 나온다.
[ ] /tf가 정상이다.
[ ] 저장된 맵이 있다.
[ ] localization이 안정적이다.
[ ] waypoint 또는 raceline 파일이 있다.
```

### 터미널 1: bringup

```bash
cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/02_bringup.sh
```

### 터미널 2: RViz2

```bash
cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/04_rviz.sh
```

### 터미널 3: localization

```bash
cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/07_localization.sh
```

여기서 RViz2에서 차량 위치가 맵 위에 맞는지 반드시 확인한다.

### 터미널 4: Pure Pursuit

```bash
cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/08_pure_pursuit.sh
```

### 동작 흐름

```text

  ↓
현재 차량 위치 추정
  ↓
pure_pursuit
  ↓
waypoint/raceline 기준 목표점 선택
  ↓
조향각 계산
  ↓
/drive 발행
  ↓
VESC
```

### 확인 명령

```bash

```

경로 관련 토픽 찾기:

```bash
ros2 topic list | grep -E "path|waypoint|goal|target|marker|pure"
```

### 주의

Pure Pursuit를 실행하기 전에 웹 대시보드가 `자율주행 모드`인지 확인하는 것이 좋다.  
웹 대시보드가 수동 모드에서 `/webop`를 계속 발행하면 주행 명령이 mux에서 꼬일 수 있다.

## 8. Stanley Avoidance 주행 묶음

Stanley Avoidance는 경로추종에 LiDAR 기반 장애물 회피를 더한 주행 방식이다.

Pure Pursuit보다 더 많은 조건이 필요하다.  
특히 `/scan` 품질과 장애물 감지 설정이 중요하다.

### 실행 묶음

```text
Bringup
  ↓
RViz2
  ↓
Localization
  ↓
Stanley Avoidance
  ├─ 경로추종
  └─ 장애물 회피
        ↓
      /drive
        ↓
      VESC
```

### 필요한 준비물

```text
[ ] 차량 bringup이 정상 동작한다.
[ ] /scan이 나온다.
[ ] /odom이 나온다.
[ ] /tf가 정상이다.
[ ] 저장된 맵이 있다.
[ ] localization이 안정적이다.
[ ] waypoint 또는 raceline 파일이 있다.
[ ] 장애물 테스트 공간이 안전하다.
```

### 터미널 1: bringup

```bash
cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/02_bringup.sh
```

### 터미널 2: RViz2

```bash

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/04_rviz.sh
```

### 터미널 3: localization

```bash

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/07_localization.sh
```

### 터미널 4: Stanley Avoidance

```bash
cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

```

### 동작 흐름

```text

  ↓
현재 차량 위치 추정
  ↓
stanley_avoidance
  ├─ Stanley 경로추종 계산
  └─ /scan으로 앞쪽 장애물 확인
        ↓

        ↓
/drive 발행
  ↓
VESC
```

### 확인 명령

LiDAR 확인:

```bash

```

drive 명령 확인:

```bash

```

```bash

```

### 주의

Stanley Avoidance 테스트에서는 사람을 장애물로 세우지 않는다.  
처음에는 박스, 콘, 스펀지 같은 충돌 위험이 낮은 물체를 사용한다.

## 9. Pure Pursuit와 Stanley를 동시에 켜면 안 되는 이유

Pure Pursuit와 Stanley Avoidance는 동시에 실행하지 않는다.

둘 다 `/drive` 명령을 발행할 가능성이 있기 때문이다.

잘못된 실행 예:

```text
pure_pursuit
  ↓
/drive

stanley_avoidance
  ↓
/drive
```

이렇게 되면 두 제어기가 동시에 차량을 제어하려고 해서 주행 명령이 꼬일 수 있다.

정상적인 선택:

```text
Pure Pursuit로 주행할 때:
  scripts/08_pure_pursuit.sh 실행

Stanley Avoidance로 주행할 때:
  scripts/09_stanley_avoidance.sh 실행
```

즉, 마지막 주행 제어기는 하나만 선택한다.

## 10. Safety Node는 어디에 들어가나?

Safety Node는 Pure Pursuit나 Stanley처럼 경로를 따라가는 주행 알고리즘이라기보다, 충돌 위험을 감지해 멈추게 하는 보호 노드에 가깝다.

개념적으로는 아래 흐름에 들어간다.

```text
/scan으로 앞쪽 거리 확인
  ↓
현재 속도와 장애물 거리 확인
  ↓
충돌 위험 계산
  ↓
위험하면 정지 명령 생성
```

mux 구조에 따라 아래처럼 연결될 수 있다.

```text
/webop
/teleop
pure_pursuit
stanley_avoidance
safety_node
  ↓
mux
  ↓
/drive
```

또는 safety node가 직접 정지 명령을 `/drive` 쪽으로 보낼 수도 있다.  
정확한 구조는 이 레포의 launch/config를 확인해야 한다.

### Safety Node 관련 파일 확인

```bash
cd ~/race_stack/f1tenth

find src/safety_node -maxdepth 5 -type f | sort
grep -R "safety\|ttc\|brake\|drive\|scan" -n src/safety_node src/f1tenth_system scripts
```

### Safety Node에서 중요한 개념

| 용어 | 의미 |
|---|---|
| TTC | Time To Collision, 충돌까지 남은 시간 |
| threshold | 위험하다고 판단하는 기준값 |
| emergency brake | 위험 시 정지 명령 |
| `/scan` | 장애물 거리 확인 |
| `/drive` | 정지 또는 주행 명령 |

### 주의

Safety Node가 있어도 완전한 안전을 보장하지 않는다.  
실차 테스트에서는 항상 낮은 속도에서 시작하고, 수동 정지 방법을 준비한다.

## 12. 어떤 묶음을 선택해야 하나?

| 상황 | 선택할 실행 묶음 |
|---|---|
| 차량 센서가 살아있는지 보고 싶다 | Bringup 묶음 |
| 직접 조작하고 싶다 | 웹 수동조작 묶음 |
| 맵이 없다 | SLAM 맵 생성 묶음 |
| 맵 위에서 위치가 맞는지 보고 싶다 | Localization 묶음 |
| 경로만 따라가고 싶다 | Pure Pursuit 묶음 |
| 경로를 따라가면서 장애물도 피하고 싶다 | Stanley Avoidance 묶음 |
| 충돌 방지 로직을 확인하고 싶다 | Safety Node 관련 launch/config 확인 |

## 13. 실차 주행 전 공통 체크리스트

Pure Pursuit든 Stanley든 실차 주행 전에는 반드시 아래를 확인한다.

```text

[ ] 주변에 사람이 없는가?
```

## 15. 핵심 정리

```text
SLAM:
  맵을 만드는 단계

Localization:
  저장된 맵에서 현재 위치를 찾는 단계

Pure Pursuit:
  현재 위치와 waypoint를 이용해 경로를 따라가는 단계

Stanley Avoidance:
  경로를 따라가면서 /scan으로 장애물을 보고 피하는 단계

Safety Node:
  충돌 위험이 있을 때 멈추게 하는 보호 단계

Bringup:
  센서, VESC, odom, TF, mux 등 차량 기본 시스템을 살리는 단계
```

실제로 주행할 때는 아래처럼 선택하면 된다.

```text
맵이 없다
→ SLAM 묶음 실행

맵이 있다, 경로만 따라가고 싶다
→ Pure Pursuit 묶음 실행

맵이 있다, 경로를 따라가면서 장애물도 피하고 싶다
→ Stanley Avoidance 묶음 실행

그냥 직접 조작하고 싶다
→ 웹 대시보드 묶음 실행
```
