# 08. 문제해결

이 문서는 F1TENTH 실험 중 자주 발생하는 문제와 확인 방법을 정리한다.

문제가 생기면 바로 코드를 수정하기보다, 먼저 아래 순서로 확인하는 것을 권장한다.

```text
1. 현재 브랜치와 git 상태 확인
2. ROS2 환경 적용 여부 확인
3. 빌드 상태 확인
4. 노드 실행 여부 확인
5. 토픽 출력 확인
6. TF 연결 확인
7. config 파일 경로 확인
8. 하드웨어 연결 확인
```

---

## 1. 기본 점검 명령어

현재 위치 확인:

```bash
pwd
```

Git 상태 확인:

```bash
cd ~/race_stack/f1tenth
git status
git log --oneline -5
```

ROS2 환경 적용:

```bash
source /opt/ros/foxy/setup.bash
cd ~/race_stack/f1tenth
source install/setup.bash
```

## 2. 빌드 문제

기본 빌드:

```bash
cd ~/race_stack/f1tenth
source /opt/ros/foxy/setup.bash
colcon build --symlink-install
source install/setup.bash
```

스크립트 사용:

```bash
bash scripts/01_build.sh
```

### `serial_driver/serial_driver.hpp` 오류

오류:

```text
fatal error: serial_driver/serial_driver.hpp: No such file or directory
```

해결:

```bash
sudo apt update
sudo apt install -y ros-foxy-serial-driver
```

### `undefined reference to SerialPort::open()` 링크 오류

확인 파일:

```text
src/f1tenth_system/vesc/vesc_driver/CMakeLists.txt
```

확인:

```bash
grep -n "serial_driver" src/f1tenth_system/vesc/vesc_driver/CMakeLists.txt
```

### 빌드 캐시 문제

```bash
cd ~/race_stack/f1tenth
rm -rf build install log
source /opt/ros/foxy/setup.bash
colcon build --symlink-install
source install/setup.bash
```

## 3. 웹 대시보드 문제

실행:

```bash
cd ~/race_stack/f1tenth
source /opt/ros/foxy/setup.bash
source install/setup.bash
python3 mission_control/dashboard.py
```

브라우저:

```text
http://localhost:5000
```

### Flask가 없다고 나오는 경우

오류:

```text
ModuleNotFoundError: No module named 'flask'
```

해결:

```bash
python3 -m pip install --user flask
```

`pip`가 없다면:

```bash
sudo apt update
sudo apt install -y python3-pip
python3 -m pip install --user flask
```

### 웹 페이지가 안 열리는 경우

```bash
ps aux | grep dashboard.py
hostname -I
```

다른 PC나 스마트폰에서 접속:

```text
http://노트북_IP:5000
```

### 버튼을 눌러도 차량이 안 움직이는 경우

```bash
ros2 topic echo /webop
ros2 topic echo /drive
```

확인할 것:

- 차량 Bringup이 실행 중인가?
- 웹 대시보드가 수동 모드인가?
- 최대 속도가 0이 아닌가?
- mux가 `/webop`를 `/drive`로 넘기고 있는가?
- VESC와 배터리가 연결되어 있는가?

## 5. VESC 문제

장치 확인:

```bash
ls /dev/ttyUSB*
ls /dev/ttyACM*
```

serial 권한 확인:

```bash
groups
```

`dialout`이 없다면 추가:

```bash
sudo usermod -aG dialout $USER
reboot
```

`/odom` 확인:

```bash
ros2 topic list | grep odom
ros2 topic echo /odom
```

config에서 포트 확인:

```bash
grep -R "ttyUSB\|ttyACM\|vesc" -n src/f1tenth_system
```

## 13. 차량이 갑자기 움직이는 경우

즉시 `정지` 버튼을 누르거나 emergency stop을 사용한다.  
그래도 멈추지 않으면 전원을 차단한다.

확인:

```bash
ros2 topic echo /drive
ros2 node list
ps aux | grep ros2
```

필요할 때만 강제 종료:

```bash
pkill -f ros2
```

주의: 실행 중인 ROS2 프로세스가 한꺼번에 종료될 수 있다.

## 14. 자주 쓰는 grep 명령

| 목적 | 명령 |
|---|---|
| launch 파일 찾기 | `find src -name "*.launch.py" | sort` |
| config 파일 찾기 | `find src -name "*.yaml" -o -name "*.yml" | sort` |
| `/drive` 사용 위치 | `grep -R "/drive" -n src mission_control scripts` |
| `/webop` 사용 위치 | `grep -R "/webop" -n src mission_control scripts` |
| `/scan` 사용 위치 | `grep -R "/scan" -n src mission_control scripts` |
| frame 이름 찾기 | `grep -R "base_link\|laser\|odom\|map" -n src mission_control scripts` |
