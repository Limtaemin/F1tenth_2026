# 08. 문제해결

이 문서는 F1TENTH 실험 중 자주 발생하는 문제와 확인 방법을 정리한다.

문제가 생기면 바로 코드를 수정하기보다, 먼저 아래 순서로 확인하는 것을 권장한다.

현재 실행 중인 ROS2 프로세스 확인:

필요할 때만 강제 종료:

pkill -f ros2

주의:

pkill -f ros2 명령은 실행 중인 ROS2 프로세스를 한꺼번에 종료할 수 있다.
다른 실험 노드도 같이 종료될 수 있으므로 주의해서 사용한다.

15. 안전 관련 주의사항

아래 토픽은 차량을 실제로 움직일 수 있다.

/drive

/teleop

실차에서 실행하기 전에는 항상 확인한다.

[ ] 주변에 사람이 없는가?
[ ] 차량 바퀴를 띄우고 먼저 테스트했는가?
[ ] emergency stop 방법을 알고 있는가?
[ ] 목표 속도가 낮은가?
[ ] /drive 명령이 예상 범위인가?
[ ] 어떤 노드가 /drive를 발행하는지 알고 있는가?

ros2 topic info /drive -v

17. 이 문서에서 바꾸지 않는 것

현재 단계에서는 아래 이름을 바꾸지 않는다.

/drive
/webop
/teleop
/scan
/odom
/map
/tf
/tf_static
base_link
laser
odom
map

이 이름들은 bringup, SLAM, localization, controller, RViz2에서 함께 사용될 수 있다.

문제해결 단계에서도 먼저 원인을 확인하고, topic/frame 이름은 함부로 바꾸지 않는다.
