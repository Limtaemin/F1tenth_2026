# 02. 수동조작과 웹조작

이 문서는 F1TENTH 차량을 직접 움직이기 위한 수동조작과 웹 대시보드 조작 방법을 설명한다.

웹 대시보드는 `mission_control/dashboard.py`에서 실행되며, 브라우저 화면에서 차량 bringup, SLAM 실행, 맵 저장, 수동 조작을 할 수 있다.

---

## 1. 전체 흐름

수동조작과 웹조작의 기본 실행 흐름은 다음과 같다.

```text
1. ROS2 환경 적용
2. 차량 bringup 실행
3. 웹 대시보드 실행
4. 브라우저 접속
5. 수동 모드 확인
6. 최대 속도 / 최대 조향 설정
7. 전진, 후진, 좌회전, 우회전, 정지 버튼으로 테스트

스크립트 기준으로는 다음 순서다.

cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/02_bringup.sh

다른 터미널에서 웹 대시보드를 실행한다.

cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

python3 mission_control/dashboard.py

브라우저에서 접속한다.

http://localhost:5000

같은 네트워크의 다른 PC나 스마트폰에서 접속할 경우에는 F1TENTH 노트북의 IP 주소를 사용한다.

http://노트북_IP:5000

예시:

http://192.168.0.23:5000
2. 실행 전 안전 체크리스트

실차에서 수동조작을 하기 전에는 반드시 아래를 확인한다.

[ ] 차량 바퀴가 바닥에서 떨어져 있는가?
[ ] 주변에 사람이 없는가?
[ ] 배터리 전압이 정상인가?
[ ] VESC가 연결되어 있는가?
[ ] LiDAR가 연결되어 있는가?
[ ] 긴급 정지 방법을 알고 있는가?
[ ] 최대 속도가 너무 높게 설정되어 있지 않은가?
[ ] 최대 조향값이 너무 크게 설정되어 있지 않은가?

처음 테스트할 때는 바퀴를 바닥에서 띄운 상태에서 진행한다.

웹 대시보드에서 버튼을 누르면 /webop 토픽으로 차량 조작 명령이 발행된다. 따라서 실차에서는 작은 속도부터 테스트해야 한다.

3. 웹 대시보드 실행

웹 대시보드는 다음 명령으로 실행한다.

cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

python3 mission_control/dashboard.py

정상 실행되면 터미널에 대략 다음과 같은 메시지가 나온다.

* Running on http://127.0.0.1:5000
* Running on http://0.0.0.0:5000

이후 브라우저에서 아래 주소로 접속한다.

http://localhost:5000
4. Flask 오류가 나는 경우

웹 대시보드 실행 시 아래 오류가 나올 수 있다.

ModuleNotFoundError: No module named 'flask'

이 경우 Flask가 설치되어 있지 않은 것이다.

아래 명령으로 설치한다.

python3 -m pip install --user flask

만약 pip가 없다면 먼저 설치한다.

sudo apt update
sudo apt install -y python3-pip

python3 -m pip install --user flask

설치 후 다시 실행한다.

cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

python3 mission_control/dashboard.py

웹 대시보드에는 다음 항목이 있다.

F1TENTH 실험 대시보드
주행 모드
차량 Bringup
SLAM / 맵
수동 조작 패드
최대 속도
최대 조향
전진
후진
좌회전
우회전
정지

각 항목의 의미는 다음과 같다.

항목설명
주행 모드수동 조작 모드와 자율주행 모드를 전환
차량 Bringup차량 하드웨어 bringup 실행/정지
SLAM / 맵SLAM 실행/정지 및 맵 저장
수동 조작 패드웹 버튼으로 차량 조작
최대 속도웹 조작에서 사용할 최대 속도 제한
최대 조향웹 조작에서 사용할 최대 조향 제한

후진속도 감소
좌회전조향값 증가
우회전조향값 감소
정지속도와 조향을 0으로 초기화
6. 수동 모드와 자율주행 모드

웹 대시보드에는 두 가지 모드가 있다.

수동 모드
자율주행 모드
수동 모드

수동 모드에서는 웹 대시보드의 버튼을 누르면 /webop 토픽으로 조작 명령이 발행된다.

전진
후진
좌회전
우회전
정지

이 모드는 사람이 직접 차량을 테스트할 때 사용한다.

자율주행 모드

자율주행 모드에서는 웹 대시보드가 /webop 조작 명령을 발행하지 않는다.

자율주행 알고리즘이 /drive 또는 관련 제어 토픽을 통해 차량을 제어할 수 있도록 하기 위한 모드다.

주의할 점:

자율주행 모드라고 해서 차량이 자동으로 출발하는 것은 아니다.
단지 웹 수동조작 명령을 멈추는 모드다.
7. 주요 토픽 관계

웹 대시보드 수동조작과 관련된 주요 토픽은 다음과 같다.

토픽역할
/webop웹 대시보드에서 발행하는 수동 조작 명령
/teleop조이스틱 또는 키보드 수동 조작 명령
/drive최종 차량 구동 명령
/scanLiDAR 스캔 데이터
/odom차량 odometry
/tf좌표계 변환

일반적인 흐름은 다음과 같다.

웹 대시보드 버튼
  ↓
/webop
  ↓
mux 또는 제어 노드
  ↓
/drive
  ↓
VESC
  ↓
모터 / 서보

즉 웹 대시보드는 직접 모터를 제어하는 것이 아니라, ROS2 토픽으로 조작 명령을 발행한다.

8. 차량 Bringup

웹 대시보드에서 차량 Bringup의 시작 버튼을 누르면 내부적으로 다음 launch가 실행된다.

ros2 launch f1tenth_stack bringup_launch.py

수동으로 실행하려면 아래 명령을 사용한다.

cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

bash scripts/02_bringup.sh

bringup이 정상적으로 실행되면 아래 토픽들이 나와야 한다.

ros2 topic list

확인할 주요 토픽:

/scan
/odom
/tf
/webop
/drive
9. 수동 조작 테스트 순서

처음 테스트할 때는 아래 순서를 권장한다.

1단계: 바퀴를 띄우고 테스트

차량 바퀴가 바닥에 닿지 않게 한다.

2단계: 웹 대시보드 실행
cd ~/race_stack/f1tenth

source /opt/ros/foxy/setup.bash
source install/setup.bash

python3 mission_control/dashboard.py
3단계: 브라우저 접속
http://localhost:5000
4단계: 수동 모드 확인

화면에서 주행 모드가 아래처럼 표시되는지 확인한다.

주행 모드: 수동 조작
5단계: 낮은 제한값 설정

처음에는 낮은 값을 사용한다.

최대 속도: 0.2
최대 조향: 0.10

값을 입력한 뒤 적용 버튼을 누른다.

6단계: 정지 버튼 먼저 확인

먼저 정지 버튼을 눌러 속도와 조향이 0이 되는지 확인한다.

7단계: 전진 / 후진 테스트

짧게 전진 버튼을 누른 뒤 바로 정지를 누른다.

그다음 후진 버튼도 같은 방식으로 테스트한다.

8단계: 좌회전 / 우회전 테스트

좌회전, 우회전 버튼을 눌러 조향이 반응하는지 확인한다.

9단계: 바닥 주행 테스트

바퀴를 바닥에 내려놓고 아주 낮은 속도로 테스트한다.

10. 토픽 확인 명령

웹 대시보드가 조작 명령을 내보내는지 확인하려면 아래 명령을 사용한다.

ros2 topic echo /webop

웹 대시보드에서 전진, 좌회전, 정지 버튼을 누르면 /webop 값이 바뀌어야 한다.

최종 drive 명령을 확인하려면 아래를 사용한다.

ros2 topic echo /drive

LiDAR가 정상인지 확인하려면:

ros2 topic echo /scan

odom이 정상인지 확인하려면:

ros2 topic echo /odom

TF가 정상인지 확인하려면:

ros2 run tf2_tools view_frames
11. 조이스틱 / 키보드 조작

이 레포에는 웹 대시보드 외에도 조이스틱 또는 키보드 조작에 관련된 파일이 포함될 수 있다.

관련 파일을 찾으려면 아래 명령을 사용한다.

cd ~/race_stack/f1tenth

find . -iname "*teleop*" -o -iname "*joy*" -o -iname "*keyboard*"

mission_control/key_teleop.py가 있다면 키보드 조작 도구로 사용할 수 있다.

실행 전에는 반드시 파일 내부의 사용법과 발행 토픽을 확인한다.

sed -n '1,220p' mission_control/key_teleop.py

주의:

키보드/조이스틱 조작도 차량을 실제로 움직일 수 있다.
처음 실행할 때는 반드시 바퀴를 띄우고 테스트한다.
12. 자주 생기는 문제
12.1 웹 페이지가 안 열린다

확인할 것:

python3 mission_control/dashboard.py

터미널에 Flask 서버가 실행 중인지 확인한다.

브라우저 주소도 확인한다.

http://localhost:5000

다른 PC에서 접속할 경우에는 localhost가 아니라 노트북 IP를 사용해야 한다.

http://노트북_IP:5000
12.2 Flask가 없다고 나온다

오류:

ModuleNotFoundError: No module named 'flask'

해결:

python3 -m pip install --user flask
12.3 버튼을 눌러도 차량이 안 움직인다

먼저 /webop가 나오는지 확인한다.

ros2 topic echo /webop

/webop가 나오는데 차량이 움직이지 않으면 /drive를 확인한다.

ros2 topic echo /drive

확인할 것:

[ ] bringup이 실행 중인가?
[ ] 웹 대시보드가 수동 모드인가?
[ ] 최대 속도가 0이 아닌가?
[ ] mux가 /webop 입력을 받고 있는가?
[ ] VESC가 연결되어 있는가?
[ ] 배터리가 연결되어 있는가?
12.4 자율주행 모드에서 버튼이 안 먹는다

정상 동작이다.

자율주행 모드에서는 웹 대시보드가 /webop 명령을 발행하지 않는다.

수동 조작을 하려면 수동 모드 버튼을 누른다.

12.5 차량이 갑자기 움직인다

즉시 정지 버튼을 누른다.

그래도 멈추지 않으면 전원을 차단하거나 emergency stop을 사용한다.

이후 아래를 확인한다.

ros2 topic echo /drive

그리고 실행 중인 노드를 확인한다.

ros2 node list

필요하면 전체 launch를 종료한다.

pkill -f ros2

주의:

pkill -f ros2 명령은 실행 중인 ROS2 프로세스를 강제로 종료할 수 있다.
실험 중인 다른 ROS2 노드도 같이 종료될 수 있으므로 주의해서 사용한다.
13. 추천 실험 순서

처음 실차 테스트를 할 때는 아래 순서를 추천한다.

1. 바퀴를 띄운다.
2. bringup을 실행한다.
3. 웹 대시보드를 실행한다.
4. /webop 토픽을 echo한다.
5. 웹 대시보드에서 정지 버튼을 누른다.
6. 최대 속도를 0.2 이하로 낮춘다.
7. 전진 버튼을 짧게 누른다.
8. 정지 버튼을 누른다.
9. 좌회전/우회전 조향만 테스트한다.
10. 문제가 없으면 바닥에서 아주 낮은 속도로 테스트한다.
14. 이 문서에서 바꾸지 않는 것

현재 단계에서는 아래 이름을 바꾸지 않는다.

/webop
/teleop
/drive
/scan
/odom
/tf
base_link
laser
odom
map

이 이름들은 여러 ROS2 노드와 launch/config 파일에서 함께 사용될 수 있다.

문서화 단계에서는 이름을 바꾸지 않고, 먼저 전체 흐름을 이해하는 것을 목표로 한다.
