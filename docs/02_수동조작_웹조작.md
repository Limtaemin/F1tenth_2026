# 02. 수동조작과 웹조작

이 문서는 F1TENTH 차량을 직접 움직이기 위한 수동조작과 웹 대시보드 조작 방법을 설명한다.

웹 대시보드는 `mission_control/dashboard.py`에서 실행되며, 브라우저 화면에서 차량 bringup, SLAM 실행, 맵 저장, 수동 조작을 할 수 있다.

---

## 1. 전체 흐름

```text
1. ROS2 환경 적용
2. 차량 bringup 실행
3. 웹 대시보드 실행
4. 브라우저 접속
5. 수동 모드 확인
6. 최대 속도 / 최대 조향 설정
7. 전진, 후진, 좌회전, 우회전, 정지 버튼으로 테스트
```

bringup 실행:

```bash
cd ~/race_stack/f1tenth
source /opt/ros/foxy/setup.bash
source install/setup.bash
bash scripts/02_bringup.sh
```

다른 터미널에서 웹 대시보드 실행:

```bash
cd ~/race_stack/f1tenth
source /opt/ros/foxy/setup.bash
source install/setup.bash
python3 mission_control/dashboard.py
```

브라우저 접속:

```text
http://localhost:5000
```

다른 PC나 스마트폰에서 접속할 경우:

```text
http://노트북_IP:5000
```

---

## 2. 실행 전 안전 체크리스트

```text
[ ] 차량 바퀴가 바닥에서 떨어져 있는가?
[ ] 주변에 사람이 없는가?
[ ] 배터리 전압이 정상인가?
[ ] VESC가 연결되어 있는가?
[ ] LiDAR가 연결되어 있는가?
[ ] 긴급 정지 방법을 알고 있는가?
[ ] 최대 속도가 너무 높게 설정되어 있지 않은가?
[ ] 최대 조향값이 너무 크게 설정되어 있지 않은가?
```

웹 대시보드에서 버튼을 누르면 `/webop` 토픽으로 차량 조작 명령이 발행된다.

---

## 3. Flask 오류가 나는 경우

오류:

```text
ModuleNotFoundError: No module named 'flask'
```

해결:

```bash
python3 -m pip install --user flask
```

`pip`가 없다면 먼저 설치한다.

```bash
sudo apt update
sudo apt install -y python3-pip
python3 -m pip install --user flask
```

---

## 4. 웹 대시보드 화면 구성

| 항목 | 설명 |
|---|---|
| 주행 모드 | 수동 조작 모드와 자율주행 모드를 전환 |
| 차량 Bringup | 차량 하드웨어 bringup 실행/정지 |
| SLAM / 맵 | SLAM 실행/정지 및 맵 저장 |
| 수동 조작 패드 | 웹 버튼으로 차량 조작 |
| 최대 속도 | 웹 조작에서 사용할 최대 속도 제한 |
| 최대 조향 | 웹 조작에서 사용할 최대 조향 제한 |
| 전진 | 속도 증가 |
| 후진 | 속도 감소 |
| 좌회전 | 조향값 증가 |
| 우회전 | 조향값 감소 |
| 정지 | 속도와 조향을 0으로 초기화 |

---

## 5. 수동 모드와 자율주행 모드

### 수동 모드

수동 모드에서는 웹 대시보드의 버튼을 누르면 `/webop` 토픽으로 조작 명령이 발행된다.

### 자율주행 모드

자율주행 모드에서는 웹 대시보드가 `/webop` 조작 명령을 발행하지 않는다.

주의:

```text
자율주행 모드라고 해서 차량이 자동으로 출발하는 것은 아니다.
단지 웹 수동조작 명령을 멈추는 모드다.
```

---

## 6. 주요 토픽 관계

| 토픽 | 역할 |
|---|---|
| `/webop` | 웹 대시보드에서 발행하는 수동 조작 명령 |
| `/teleop` | 조이스틱 또는 키보드 수동 조작 명령 |
| `/drive` | 최종 차량 구동 명령 |
| `/scan` | LiDAR 스캔 데이터 |
| `/odom` | 차량 odometry |
| `/tf` | 좌표계 변환 |

일반적인 흐름:

```text
웹 대시보드 버튼
  ↓
/webop
  ↓
mux 또는 제어 노드
  ↓
/drive
  ↓
VESC
  ↓
모터 / 서보
```

---

## 7. 수동 조작 테스트 순서

1. 차량 바퀴가 바닥에 닿지 않게 한다.
2. 웹 대시보드를 실행한다.
3. 브라우저에서 `http://localhost:5000`에 접속한다.
4. 주행 모드가 `수동 조작`인지 확인한다.
5. 최대 속도는 `0.2` 이하, 최대 조향은 `0.10` 정도로 시작한다.
6. 먼저 `정지` 버튼을 누른다.
7. `전진`을 짧게 누른 뒤 바로 `정지`를 누른다.
8. `좌회전`, `우회전` 반응을 확인한다.
9. 문제가 없으면 바닥에서 낮은 속도로 테스트한다.

---

## 8. 토픽 확인 명령

```bash
ros2 topic echo /webop
ros2 topic echo /drive
ros2 topic echo /scan
ros2 topic echo /odom
ros2 run tf2_tools view_frames
```

---

## 9. 자주 생기는 문제

### 웹 페이지가 안 열린다

- `python3 mission_control/dashboard.py`가 실행 중인지 확인한다.
- 같은 노트북이면 `http://localhost:5000`으로 접속한다.
- 다른 PC나 스마트폰이면 `http://노트북_IP:5000`으로 접속한다.

### 버튼을 눌러도 차량이 안 움직인다

확인할 것:

```text
[ ] bringup이 실행 중인가?
[ ] 웹 대시보드가 수동 모드인가?
[ ] 최대 속도가 0이 아닌가?
[ ] mux가 /webop 입력을 받고 있는가?
[ ] VESC가 연결되어 있는가?
[ ] 배터리가 연결되어 있는가?
```

### 자율주행 모드에서 버튼이 안 먹는다

정상 동작이다. 수동 조작을 하려면 `수동 모드` 버튼을 누른다.
